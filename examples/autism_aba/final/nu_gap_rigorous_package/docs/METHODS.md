# METHODS

## 1) Data and scope
We used session‑by‑session points digitized from Figure 1 (formatting verification requirement). Each row includes:
- participant, metric_name, session_index, metric_value, phase, ioa, caregiver_fidelity, researcher_fidelity, gap_flag.
The primary metric here is **IDKPTM‑Set1**. IOA/fidelity were taken from Table 1 (means).

## 2) Lean N/U Core v1 (brief)
Represent each observation as (n,u), where n∈[0,1] is the nominal signal (metric_value) and u≥0 is uncertainty derived from IOA and fidelity.
Operations: (n1,u1)⊕(n2,u2)=(n1+n2,u1+u2); (n1,u1)⊗(n2,u2)=(n1n2, |n1|u2+|n2|u1); a⊙(n,u)=(an, |a|u).
Diagnostics: Catch Cα(n,u)=(0,|n|+u) and Flip B(n,u)=(u,|n|), preserving M=|n|+u.

## 3) Weighting
Per session uncertainty u_t = (1−IOA_t) + (1−Caregiver_t) + (1−Researcher_t), clipped to [0,1].
Weights w_t = 1/(u_t + ε).

## 4) Statistics
Within a participant, compare **pre** vs **post** blocks defined by gap_flag.
- Level shift: Δ_L = mean_post(w) − mean_pre(w).
- Trend shift: Δ_T = slope_post(w) − slope_pre(w), where slope is WLS of metric_value on session_index with weights w_t.
- SESOI: |Δ_L| ≤ 0.10 and |Δ_T| ≤ 0.005/session considered practically negligible.

## 5) Permutation inference
Null is generated by circularly shifting pre/post labels while preserving counts (design‑respecting). For each permutation b, compute Δ_L^(b), Δ_T^(b). Two‑sided p‑values are:
p = (1 + #{|Δ^(b)| ≥ |Δ_obs|}) / (1 + B).

## 6) Reporting
We report Δ_L, Δ_T, permutation p‑values, counts (n_pre, n_post), and an SESOI verdict. Sensitivity runs can invert the metric or include 5‑targets sessions.
