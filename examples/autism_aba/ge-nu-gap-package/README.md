# GE‑NU Gap‑Effect Analysis Package
**Purpose:** Reproduce a quantitative test that the **between‑session gaps** in Swensson et al. (2024) did **not** affect outcomes, using a conservative **Nominal/Uncertainty (N/U)** weighting.

**Lead artifact owner:** Eric D. Martin  
**Created:** 2025-10-02

## What’s inside
- `docs/METHODS.md` — Plain‑language intro to N/U algebra + the exact gap‑effect test.
- `docs/LIMITATION_EXCERPT.md` — Short, properly cited excerpt where Swensson et al. note the gap limitation.
- `docs/REFERENCES.md` — Sources (with DOIs) for the article and N/U algebra paper.
- `data/sessions_template.csv` — **Fill this** with per‑session data for each participant.
- `scripts/ge_nu_gap_effect.py` — One‑file reproducible analysis (permutation test + uncertainty‑weighted slopes/levels + decision table).
- `results/` — Empty folder for CSV/MD outputs generated by the script.

## Quick start
1. Enter per‑session data into `data/sessions_template.csv` (one row per session).  
2. Run the analysis:
   ```bash
   python scripts/ge_nu_gap_effect.py --csv data/sessions_template.csv --metric IDKPTM --sesoi_level 0.10 --sesoi_slope 0.005 --perms 10000
   ```
3. See outputs in `results/`:
   - `gap_effect_summary.csv` — per‑participant Δ_L, Δ_T, permutation p‑values.
   - `decision.md` — human‑readable verdict + notes on sensitivity.

## Reproducibility knobs
- **Weights:** w_t = 1/(u_t + ε). u_t = 1 − IOA + fidelity penalties (clipped [0,1]).
- **Primary metric:** choose `IDKPTM` or `Correct`; re‑run with the inverse as sensitivity.
- **Equivalence band (SESOI):** ±10% level; ±0.005/session slope (defaults; change via flags).
- **Permutation test:** randomized cut‑point respecting the observed count of pre/post sessions within intervention.

## What this package does *not* do
- It does **not** digitize the figure for you. You must enter the per‑session values from Figure 1 or your raw sheet.
- It does **not** redistribute the Wiley PDF. We cite DOI and quote ≤25 words (fair use).
