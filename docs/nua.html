Abstract

Quantifying and propagating uncertainty is central to every branch of science and engineering. Any claim without explicit error bounds risks being false, misleading, or unsafe. Existing approaches force trade-offs: Gaussian propagation is efficient but assumption-sensitive; Monte Carlo is general but costly and hard to audit; interval arithmetic is rigorous but often over-pessimistic.

We propose Nominal/Uncertainty (N/U) Algebra: a minimal algebra where every quantity is a pair $(n, u)$ of nominal value and explicit nonnegative uncertainty. The operations

$$(n_1, u_1) \oplus (n_2, u_2) = (n_1 + n_2, u_1 + u_2)$$
$$(n_1, u_1) \otimes (n_2, u_2) = (n_1 n_2, |n_1| u_2 + |n_2| u_1)$$
$$a \odot (n, u) = (a n, |a| u)$$

are closed, monotone, and run in constant time. Special operators "catch" and "flip" preserve a conservation invariant $M = |n| + u$.

We prove closure, associativity, commutativity, sub-distributivity, and cumulative-product formulas. Worked examples compare N/U against Gaussian error propagation, Monte Carlo, and interval arithmetic, showing that N/U bounds are always conservative yet remain tractable. By pinning uncertainty at Layer 2 of a Seven-Layer methodology, N/U provides a transparent, reproducible baseline for regulated workflows and safety-critical decisions.

1. Introduction

Every scientific measurement is incomplete. A length, a voltage, a gene frequency—all have finite resolution and context. Reporting only a nominal is dangerous: it suggests a false precision. Conversely, overstating uncertainty paralyzes decision-making.

Classical methods each take a stance. Gaussian or "root-sum-of-squares" rules are tight if errors are independent and small, but collapse when correlations or non-Gaussian tails intrude (Taylor, 2022). Monte Carlo simulates thousands of draws, robust in principle but computationally costly and hard to audit without fixed seeds (Metropolis & Ulam, 1949). Interval arithmetic encloses every possibility, guaranteeing truth but exploding in width under multiplication (Moore et al., 2009).

What is needed is a baseline: algebraic, assumption-free, conservative, and fast. N/U Algebra was designed to fill this role. It is lean enough for spreadsheets and embedded systems, rigorous enough for regulators, and transparent enough for auditors.

This manuscript presents the formal system, proves its key properties, demonstrates worked examples, and situates it in modern reproducibility frameworks like the Seven-Layer Methodology.

2. Preliminaries and Notation

We define the domain $A = \mathbb{R} \times \mathbb{R}_{\geq 0}$. An element $x = (n, u)$ carries a nominal $n$ and an uncertainty radius $u \geq 0$.

Partial order: $(n, u) \preceq (n, u')$ if $u \leq u'$.

Invariant: $M(n, u) = |n| + u$, used to prove conservation.

Example 2.1 (measurement). A caliper reads $25.40 \pm 0.02$ mm: $x = (25.40, 0.02)$. Another part $12.70 \pm 0.01$ mm: $y = (12.70, 0.01)$. These will be inputs for addition and multiplication rules.

3. Algebraic Operations

For $x = (n_1, u_1), y = (n_2, u_2) \in A$ and scalar $a \in \mathbb{R}$:

$$x \oplus y = (n_1 + n_2, u_1 + u_2)$$
$$x \otimes y = (n_1 n_2, |n_1| u_2 + |n_2| u_1)$$
$$a \odot x = (a n_1, |a| u_1)$$

Special operators:

$$C(n, u) = (0, |n| + u)$$
$$B(n, u) = (u, |n|)$$

Example 3.1 (addition). $(25.40, 0.02) \oplus (12.70, 0.01) = (38.10, 0.03)$ mm.

Example 3.2 (multiplication, positive factors). $(10, 1) \otimes (20, 2) = (200, 40)$.

Example 3.3 (multiplication, signs). $(-3, 0.4) \otimes (2, 0.5) = (-6, 2.3)$.

Example 3.4 (scalar). $(-4) \odot (3, 0.2) = (-12, 0.8)$.

Example 3.5 (catch/flip). $C(7, 0.6) = (0, 7.6)$; $B(-2, 0.5) = (0.5, 2)$.

4. Properties and Proofs

We list fundamental properties with concise proofs and examples.

Closure. All operations map $A \to A$. Uncertainty terms are sums of nonnegatives.

Identities. $(0, 0)$ is additive; $(1, 0)$ multiplicative.

Commutativity. Immediate from symmetry of addition and multiplication.

Associativity of addition. Standard real addition.

Associativity of multiplication. For $x = (a, A), y = (b, B), z = (c, C)$:

$$(x \otimes y) \otimes z = (abc, |ab|C + |ac|B + |bc|A)$$
$$x \otimes (y \otimes z) = (abc, |ab|C + |ac|B + |bc|A)$$

Example 4.1. With $x = (3, 0.1), y = (4, 0.2), z = (2, 0.05)$: nominal $24$; uncertainty $3.6$, same whichever grouping is used.

Monotonicity. Each rule is linear in nonnegative $u_i$; increasing any $u_i$ cannot decrease the output.

Sub-distributivity. $x \otimes (y \oplus z) \preceq (x \otimes y) \oplus (x \otimes z)$, since $|b + c| \leq |b| + |c|$.

Conservation. $M(C(n, u)) = M(B(n, u)) = |n| + u$.

Cumulative product.

$$\bigotimes_{i=1}^m (n_i, u_i) = \left(\prod_i n_i, \sum_{i=1}^m u_i \prod_{j \neq i} |n_j|\right)$$

Proof. Induction on $m$.

Function lifting. If $f$ is $L$-Lipschitz, define $f^\sharp(n, u) = (f(n), Lu)$. Then $|f(n + \delta) - f(n)| \leq L|\delta|$ ensures conservatism.

5. Related Work

Standards and engineering practice. The Joint Committee for Guides in Metrology's Guide to the Expression of Uncertainty in Measurement (GUM) remains the canonical reference for reporting and propagating measurement uncertainty; the NIST/SEMATECH e-Handbook provides practical engineering treatments and worked propagation examples. N/U aligns with this ecosystem by offering a conservative, algebraic baseline that is assumption-free and easy to audit.

Probabilistic methods. The Gaussian (delta / RSS) approach is efficient and tight under small-error and independence assumptions; Monte Carlo offers assumption-flexible uncertainty estimates at sampling cost $O(N)$ and requires careful seeding for reproducibility. N/U is intentionally distribution-free and serves as a conservative ceiling when such assumptions are not justified.

Deterministic enclosure. Interval arithmetic guarantees inclusion, but multiplicative chains can widen markedly via dependency; N/U avoids the interval product's pessimism while remaining auditable.

Spectral approaches. Polynomial chaos expansions (PCE) deliver spectral accuracy when the law/regularity is known; coefficient growth with order and dimension underpins the common $O(p^d)$ shorthand. N/U complements PCE by providing a distribution-free safety ceiling.

Imprecise probability and evidence. Probability boxes (p-boxes) and Dempster–Shafer structures bound distributions and support evidence combination, typically with higher modeling and computational overhead than N/U's pointwise bounds.

Vagueness. Fuzzy sets and $\alpha$-cut arithmetic model graded membership (vagueness), not randomness. These tools have distinct semantics and are complementary to N/U's conservative-radius calculus.

6. Results

6.1 Preliminary Rules and Corrections (transparent record)

Early experiments used a non-absolute product and an involutive flip:

$$(n_1, u_1) \otimes (n_2, u_2) = (n_1 n_2, n_1 u_2 + n_2 u_1)$$
$$B(n, u) = (u, n)$$

Both fail closure for valid inputs with negative nominals:

$$(-1, 0.1) \otimes (0, 0.2) = (0, -0.2)$$
$$B(-2, 0.5) = (0.5, -2)$$

We therefore adopt the closure-preserving rules:

$$(n_1, u_1) \otimes (n_2, u_2) = (n_1 n_2, |n_1| u_2 + |n_2| u_1)$$
$$B(n, u) = (u, |n|)$$

These restore closure, preserve monotonicity, and conserve $M(n, u) = |n| + u$. Flip ceases to be an involution—a conscious trade for well-typedness. All computations below use the corrected rules.

Note. A typographic slip in §4's three-factor "sanity check" produced $0.8 + 1.2 + 0.6 = 3.6$; the correct sum is $2.6$.

6.2 Worked computations (with cross-method comparisons)

R1 (sum chain). $(10 \pm 1) \oplus (5 \pm 0.5) = (15, 1.5)$.
RSS: $\sqrt{1^2 + 0.5^2} = 1.118$. MC ($N$ large): $\approx 1.12$. Intervals: half-width $1.5$. N/U matches interval-sum and overbounds RSS/MC as designed.

R2 (product). $(10 \pm 1) \otimes (5 \pm 0.5) = (50, |10| \cdot 0.5 + |5| \cdot 1) = (50, 10)$.
RSS: $\sqrt{(10 \cdot 0.5)^2 + (5 \cdot 1)^2} = 7.07$. Intervals $[9, 11] \times [4.5, 5.5] \Rightarrow$ half-width $10$. N/U equals interval half-width here and cleanly overbounds RSS.

R3 (ten identical factors). $\bigotimes_{i=1}^{10} (1.00 \pm 0.01) = (1.000, 0.10)$ by the cumulative formula. Intervals $[0.99, 1.01]^{10} \Rightarrow [0.99^{10}, 1.01^{10}]$ with half-width $\approx 0.1005$. RSS (small-error, independent) gives $\sqrt{10} \cdot 0.01 \approx 0.0316$.

R4 (three-factor product). $(3 \pm 0.1) \otimes (4 \pm 0.2) \otimes (2 \pm 0.05) = (24, 0.1 \cdot 8 + 0.2 \cdot 6 + 0.05 \cdot 12) = (24, 2.6)$.
Intervals $[2.9, 3.1] \times [3.8, 4.2] \times [1.95, 2.05] \Rightarrow$ half-width $\approx 2.601$. RSS $\approx 1.562$. Again: RSS < Interval-half-width ≈ N/U.

7. Applications

A1 (Pedagogical chain). Three parts measured $25.40 \pm 0.02$ mm each convert to cm via $(0.1, 0)$: each $(2.540, 0.002)$ cm; sum $(7.620, 0.006)$ cm. This style mirrors GUM/NIST record-keeping and is ideal for instructional labs and QA spreadsheets.

A2 (Engineering multiplicative cascade). Gear-train ratio $R = \prod r_i$ with $r_1 = (1.50, 0.02), r_2 = (0.80, 0.01), r_3 = (1.20, 0.02)$ gives

$$n = 1.44, \quad u = 0.02 \cdot |0.96| + 0.01 \cdot |1.80| + 0.02 \cdot |1.20| = 0.0612$$

Intervals and RSS give alternative bounds under different assumptions; N/U supplies the conservative audit trail.

A3 (High-dimensional screen). For $d = 20$ factors $(1.00 \pm 0.005)$, Prop. P7 yields $u = 0.10$. Monte Carlo approximates distributional features at $O(N)$ cost; PCE is sharper under structure but grows with basis size. N/U is a fast ceiling before invoking MC or PCE.

8. Comparative Analysis

When to use which tool.

  • N/U (this work): zero assumptions, $O(1)$ primitives, reproducible and audit-ready; conservative by design.
  • Gaussian/RSS: tight if small-error/independent; optimistic if violated.
  • Monte Carlo: flexible and asymptotically correct; runtime and seed management matter.
  • Intervals: enclosure guaranteed; dependency blow-up possible.
  • PCE / spectral: spectral accuracy under regularity/known law; coefficient growth with order/dimension.
  • P-boxes / DS: rigorous distributional bounds; higher modeling cost.
  • Fuzzy: graded membership (vagueness) via $\alpha$-cuts; different semantics than randomness.

9. Complexity and Reproducibility

Complexity. Each primitive op is $O(1)$; an $m$-term chain is $O(m)$ using the cumulative formula (Prop. P7). Monte Carlo scales with sample size; PCE's coefficient count grows with order and dimension (hence $p^d$ folklore).

Reproducibility plan. Provide a versioned artifact (code/data), environment lock, and fixed seeds for MC; include one-command runners that regenerate Table 1 and the application numbers from declarative configs. This mirrors GUM/NIST guidance for auditable calculation and documentation chains.

10. Conclusion

N/U Algebra offers a minimal, closed, and auditable calculus for uncertainty. It is not a replacement for probabilistic or spectral methods; it is a safe ceiling—fast, transparent, and distribution-free. In safety-critical or regulated work, such a ceiling is essential; when assumptions are justified, Gaussian/RSS, Monte Carlo, or PCE can tighten bounds beneath it. The worked examples and comparisons demonstrate that N/U reliably overbounds RSS/MC while avoiding interval-product explosion, and the proofs ensure the algebra behaves predictably in composition. Coupled with a straightforward reproducibility plan, N/U provides a practical foundation for trustworthy quantitative claims.

Appendix A — Expanded Derivations (selected)

A.1 Cumulative product (full induction).

Base $m = 2$ is the definition. Assume for $m$. Let $\Pi_m = \prod_{i=1}^m n_i$. Then

$$\left(\Pi_m, \sum_{i=1}^m u_i \prod_{j \neq i, j \leq m} |n_j|\right) \otimes (n_{m+1}, u_{m+1}) = \left(\Pi_m n_{m+1}, |\Pi_m| u_{m+1} + \sum_{i=1}^m u_i \prod_{j \neq i, j \leq m} |n_j| |n_{m+1}|\right)$$

which equals $\left(\prod_{i=1}^{m+1} n_i, \sum_{i=1}^{m+1} u_i \prod_{j \neq i} |n_j|\right)$.

A.2 Sub-distributivity details.

For $x = (a, A), y = (b, B), z = (c, C)$, the nominal is $a(b + c)$ both sides; radii compare via

$$|b + c| \leq |b| + |c|, \text{ hence } x \otimes (y \oplus z) \preceq (x \otimes y) \oplus (x \otimes z)$$

A.3 Function lifting and composition.

If $f$ is $L_f$-Lipschitz near $n$ and $g$ is $L_g$-Lipschitz near $m$, then

$$(f \circ g)^\sharp(n, u) = (f(g(n)), L_f L_g u)$$

by successive application of the Lipschitz bounds.
